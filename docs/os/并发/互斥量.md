# 互斥量（Mutex）

经常被叫做互斥锁

相比于自旋锁，线程对互斥锁加锁失败就**休眠**，让出 CPU 资源，当锁的状态发生改变时再**唤醒**。


### 缺点

因为操作系统负责线程调度，为了实现「锁的状态发生改变时再唤醒」就需要把锁也交给操作系统管理。所以互斥锁的加锁操作通常会需要涉及到用户态到内核态的转换，线程挂起和唤醒，以及线程上下文切换，开销也比自旋锁要大。
